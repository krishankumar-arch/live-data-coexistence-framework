# LDCF End-to-End Data Flow

> Sequence diagram showing a complete CDC event lifecycle

```mermaid
sequenceDiagram
    autonumber
    participant LEG  as Legacy MySQL
    participant DMS  as AWS DMS
    participant S3   as Amazon S3
    participant L1   as Lambda 1<br/>Claim Check
    participant SQS  as Amazon SQS
    participant L2   as Lambda 2<br/>Upstream Loader
    participant L3   as Lambda 3<br/>Metadata Service
    participant AUR  as Aurora PostgreSQL
    participant DDB  as DynamoDB Audit
    participant DLQ  as SQS DLQ
    participant L4   as Lambda 4<br/>DLQ Requeue

    Note over LEG,L4: Happy Path — Successful CDC Event

    LEG->>DMS: INSERT INTO claim (claim_id=2000000, ...)
    DMS->>S3: Write CDC_TXN_20260615.csv (pipe-separated)
    S3->>L1: S3 ObjectCreated event
    L1->>S3: Read first row only (Range: bytes=0-2047)
    L1->>SQS: Send Claim Check pointer<br/>{metadata:{bucket,key}, data:{op,table,schema}}<br/>MessageAttr: retry_count=0, is_retry=false
    SQS->>L2: Trigger (batch size=1)
    L2->>L3: Invoke metadata service<br/>{src_schema:insurance, src_table:claim}
    L3->>AUR: SELECT map_master + rules WHERE src_tbl=claim
    AUR-->>L3: 2 mappings (claim→claim, claim→claim_keyring)
    L3-->>L2: {mappings:[{map_id:3,rules:[...]},{map_id:4,rules:[...]}]}
    L2->>S3: GetObject full CSV
    Note over L2: Apply Pattern 2: datasync_flag=NULL → proceed
    Note over L2: Build dynamic DML from metadata_rules
    L2->>AUR: BEGIN; INSERT INTO insurance.claim<br/>ON CONFLICT DO UPDATE; COMMIT;
    AUR-->>L2: OK — 1 row affected
    L2->>DDB: Write SUCCESS {event_metadata, event_data, ttl=90d}
    L2->>AUR: BEGIN; INSERT INTO insurance.claim_keyring<br/>(legacy_claim_num); COMMIT;
    AUR-->>L2: OK — UUID generated by gen_random_uuid()
    L2->>DDB: Write SUCCESS {map_id:4, ttl=90d}

    Note over LEG,L4: Failure Path — DLQ Retry Flow

    L2->>AUR: BEGIN; INSERT INTO insurance.claim; COMMIT;
    AUR-->>L2: ERROR — connection timeout
    L2->>L2: Rollback transaction
    L2->>DDB: Write FAILED {retry_count=1, error, ttl=180d}
    L2->>DLQ: Send event {retry_count=1, is_retry=true}
    DLQ->>L4: Trigger
    L4->>L4: Check retry_count=1 < MAX_RETRIES=3
    L4->>SQS: Requeue {retry_count=2, is_retry=true}
    SQS->>L2: Trigger (retry attempt 2)
    L2->>AUR: BEGIN; INSERT ...; COMMIT;
    AUR-->>L2: OK
    L2->>DDB: Update to SUCCESS
```

## SQS Message Structure

```json
{
  "metadata": {
    "bucket":    "ldcf-poc-cdc-landing-123456789",
    "key":       "dms/sync/CDC_TXN_20260615143022.csv",
    "timestamp": "2026-06-15T14:30:22Z",
    "file_size": 4096
  },
  "data": {
    "event_id":  "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "operation":  "I",
    "src_table":  "claim",
    "src_schema": "insurance",
    "event_ts":   "2026-06-15T14:30:20Z"
  }
}
```

**MessageAttributes (separate from payload — payload never modified):**
```
retry_count : Number : 0
is_retry    : String : false
```

## CSV File Format

Pipe-separated, **no header row**. Column positions are fixed:

| Position | Name | Example |
|----------|------|---------|
| COLUMN1 (pos 0) | Operation | `I` / `U` / `D` |
| COLUMN2 (pos 1) | Source table | `claim` |
| COLUMN3 (pos 2) | Source schema | `insurance` |
| COLUMN4 (pos 3) | Event timestamp | `2026-06-15 14:30:20` |
| COLUMN5+ (pos 4+) | Data fields | Mapped by `column_order` in `metadata_rules` |

Example row:
```
I|claim|insurance|2026-06-15 14:30:20|CLM-2024-0001|2000000|POL-2024-0001|...
```
